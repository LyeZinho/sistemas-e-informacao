Sistemas

Um sistema é um conjunto de organismos
que trabalham em conjunto para um objetivo.

















Um exemplo é o sistema digestivo
que trabalham em conjunto com varios orgãos
para realizar a digestão de alimentos no corpo
humano.

















O sistema nervoso pode ser considerado
um subsistema, pois trabalha em conjunto com
outros orgãos dentro de um sistema maior que é o corpo
humando.

























Subsistemas

Um subsistema é quando um sistema menor
opera dentro de um sistema maior.



















Dados

Um dado é uma representação de um objeto, fato, pessoa
ou outros tipos de entidades ou acontecimentos podem essas
representações podem ser símbolos, números,
palavras ou outros tipos de sinais.








Informação

Uma informação é um conjunto de de dados organizados
e sujeitos a tratamento assim podendo ser usados em
qualquer contexto um dado não tem qualquer valor e só se
transforma em informação quando relacionados.












Uma informação atual e correta só é possível se os seus dados estiverem atualizados e forem precisos. De outra forma, a nossa informação não será útil.





Mas, para que isto possa acontecer, existem algumas condições que os dados devem garantir.

  • Atualidade
  • Correção
  • Relevância
  • Disponibilidade
  • Legibilidade




Para as organizações e os seus sistemas de informação, os dados são vistos como registros em bruto, ou seja, antes de serem tratados e organizados. Por sua vez, a informação é constituída por dados depois de tratados e organizados de forma significativa para os seres humanos.

Para uma empresa ou outro tipo de organização, a informação refere-se a objetos ou acontecimentos com importância para o funcionamento dessa organização, tais como: mercadorias, compras, vendas, pessoas, processos de trabalho, etc.










Sistemas de Informação

Conjunto de recursos utilizados para recolher e
tratar os dados e a informação que são importantes para
a atividade de uma organização.












De uma forma simples, podemos dizer que um SI para uma organização é o conjunto integrado de recursos humanos e tecnológicos, cujo objetivo é satisfazer adequadamente as necessidades da organização em termos de informação.








Para satisfazer adequadamente as necessidades de informação de uma organização. A concretização deste objetivo, passa por diferentes tipos de atividades como

  • Recolher dados e informação
  • Processar e armazenar dados
  • Distribuir e disponibilizar informação

  • O sistema de informação e composto por elementos como.


  • Recursos humanos
  • Recursos tecnológicos
  • Dados e informação
  • Processos de trabalho













Uma maneira muito divulgada de olhar para os diferentes tipos de sistemas de informação das organizações consiste em representar num triângulo os 3 principais níveis hierárquicos que existem numa empresa e identificar os principais tipos de SI que existem em cada um desses níveis.



Nivel operacional


Dizem respeito às operações do dia a dia da empresa relativas à produção (empresa industrial) ou à prestação de serviços (empresa de serviços), bem como à atividade comercial de compras e vendas, contabilidade, etc.

Nivel tático


Tem a ver com as atividades típicas dos gestores de nível intermediário, relativas, por exemplo, à análise das vendas, controle da qualidade, controle orçamental, etc.

Nivel estratégico


Diz respeito aos altos dirigentes da empresa (administradores e gestores de topo) e às decisões mais importantes para o futuro da empresa, tais como: compras de outras empresas, ampliação ou criação de novas instalações, remodelação do modo de funcionamento, etc.

Base de dados










Porque uma base de dados?



  • Permite armazenar dados de diversos tipos.

  • Permite acesso rapido e efficiente de dados.

  • Acelera processos de tratamento de dados.

  • Permite tratamente de grandes quantidades de dados

  • Economiza toneladas de papel.














Base de Dados


Sistema de armazenamento de dados relacionados entre si, de uma forma permanente, num sistema informático, com redundância controlada, acessíveis a um grupo de utilizadores e estruturado sob a forma de ficheiros de dados ou tabelas.







Base de Dados oque é?


É uma coleção lógica e coerente de dados, com um significado inerente;

Uma base de dados é planeada, construída e atualizada com dados que representam aspetos do mundo real, como se de um “pequeno mundo se tratasse”.

Para uma base de dados deve existir um Administrador. Este é o responsável máximo da base de dados.







Tempo de vida de uma base de dados


É a expressão utilizada para designar todos os
eventos que acontecem desde a primeira vez que
é reconhecida a necessidade de uma base de
dados, até ao seu desenvolvimento,
implementação e eliminação.









Apenas podemos passar para a fase seguinte depois da anterior estar concluída;

Por vezes, surge a necessidade de retroceder à fase anterior, para realizar determinados ajustes.











1 Planeamento

Levantamento das necessidades, organização e planeamento.




2 Recolha de requisitos

Elaboração de um documento com os objetivos que o projeto visa atingir.




3 Desenho Concetual

Desenho de todos os modos de vista externos da aplicação e da base de dados. O aspeto dos formulários, relatórios, ecrãs de entrada de dados, etc.




4 Desenho lógico

A partir do desenho conceptual cria-se o desenho lógico da aplicação.







5 Desenho físico

Durante esta fase, o desenho lógico é mapeado ou convertido para os sistemas de software que serão usados na implementação da aplicação e da base de dados.




6 Construção

As unidades de programação são promovidas para um sistema de ambiente de teste, onde toda a aplicação e base de dados é montada e testada.




7 Implementação

Instalação e colocação em funcionamento da nova aplicação e base de dados.




8 Manutenção

Resolver quaisquer situações de anomalias ou erros, normalmente designados por “bugs”, quer ao nível da aplicação, quer ao nível da base de dados.
















SDGBD

São programas que permitem usualmente, trabalhar com múltiplas tabelas e/ou outras unidades de dados ou informação, estabelecer relacionamentos entre essas entidades e efetuar muitas outras operações de organização da informação que são específicas deste tipo de programas.

















Caracteristicas

A independência dos dados - relativamente aos programas e procedimentos predefinidos que os trabalham

Redundância controlada - atendendo a que existe utilização comum dos dados, a redundância da informação não deve existir. Podem existir exceções, em determinadas situações, para tornar mais rápido o acesso aos dados

Integridade dos dados - esta é assegurada por procedimentos predefinidos de validação dos dados sempre que se trate de uma atualização
